# Daily Hike Recommendation System

Trail-Intel ranks the seven supplied Lebanon GPX trails deterministically. The AI briefing is not used for filtering or scoring.

## Pipeline

1. Validate the request and active trail catalog.
2. Estimate drive time through `DriveTimeProvider`. The current provider converts great-circle distance to a 1.35× road-distance estimate at 45 km/h.
3. Simulate hiking time from distance, ascent, profile pace, and pack allowance using the same inputs as the existing trail page.
4. Attach Open-Meteo hourly apparent temperature, rain probability, wind, and daily sunset in `Asia/Beirut`.
5. Apply hard exclusions before ranking: unavailable trail, drive limit, hike-time limit, excluded feature, confidence below 60, finish more than 15 minutes after sunset, rain ≥85%, or wind ≥60 km/h.
6. Score remaining and excluded trails using configurable weights: fitness 30%, weather 20%, daylight 15%, preferences 15%, logistics 10%, confidence 10%. Scores are normalized to 0–100.
7. Sort eligible trails first, then overall score, then trail ID for deterministic ties. Assign Best Match, More Adventurous, and Easier Alternative when suitable eligible trails exist.

Every result exposes positive and negative evidence factors, hard-exclusion codes, weather/trail timestamps, `analysisVersion`, `scoringVersion`, and confidence.

## Sample request

```json
{"date":"2026-07-25","preferred_departure_time":"07:00","starting_location":{"latitude":33.8938,"longitude":35.5018},"maximum_drive_minutes":90,"available_total_hours":8,"preferred_difficulty":["moderate"],"preferred_features":["forest","viewpoint"],"avoid_features":["high_exposure"],"profile":{"distanceKm":18,"ascentM":1200,"paceKmh":4.5,"packKg":6}}
```

`POST /api/recommendations/daily` returns `groups`, the full `ranked` list, excluded trails, generation time, timezone, and `partialData`. `GET /api/recommendations/{recommendation_id}` retrieves a result generated by the current server instance.

## Assumptions and mocked services

- Starting coordinates default to central Beirut and remain user-editable; browser geolocation is not requested.
- Drive time is an explicit distance-based fallback, not road routing. Replace `DriveTimeProvider` with Mapbox or Google routing later.
- Trail difficulty/features/exposure are catalog metadata inferred for the hackathon and should later be editorially reviewed.
- Water is reported only where catalog metadata contains `water`; shelters/access points are currently unknown and reduce explanatory certainty rather than being invented.
- Trail closure data is unavailable. All seven permission-granted recordings are marked active.
- Weather failures never silently reuse stale data. Results are labeled partial, use conservative component values, and expose a null weather timestamp.
- Recommendation lookup is in-memory and expires when the server restarts; no database migration is required for the current request-driven demo.
