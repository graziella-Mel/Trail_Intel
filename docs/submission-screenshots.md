# Submission Screenshot Checklist

Capture at 1920×1080 unless the submission form specifies otherwise. Use the public URL, a clean browser session, and no personal browser UI. Keep text large enough to read in a thumbnail.

1. **Landing page** — Trail-Intel name, selected Lebanon trail, 3D map, route signal metrics, and “Try Hackathon Demo” must be visible.
2. **Daily Hike Recommendations** — planning inputs, Best Match score, and all three ranked Lebanon recommendations must be visible. Include the reason lines that distinguish the alternatives.
3. **Best Match details** — Zaarour → Mtein → Mrouj title, readiness, distance, ascent, predicted duration, and “Continue to Live Hike” must be visible.
4. **Trail map and elevation analysis** — route geometry, fatigue legend, highlighted struggle section, and elevation/effort chart must be visible together or as a clean two-image pair.
5. **Fatigue visualization** — the red highest-fatigue segment, “Likely struggle point,” kilometer, score, and ascending/descending label must be readable.
6. **Live Hike dashboard** — active state, progress, actual versus planned pace, updated finish, daylight, fatigue, and map position must be visible.
7. **Adaptive warning** — “Pace Behind Plan,” supporting numeric evidence, next higher-effort section, and the structured recommendation must be visible.
8. **Off-route detection** — run “Off-Route Deviation” and capture route status, distance from route, warning evidence, and recovery guidance.
9. **Post-hike summary** — completed state, actual/moving/stopped duration, maximum deviation, warning count, final predicted finish, and optional profile-review suggestion must be visible.

Before upload, verify that no email address, local filesystem path, API credential, developer overlay, or localhost URL appears in any screenshot.
